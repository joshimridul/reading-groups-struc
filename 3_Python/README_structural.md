# Structural Extension Pipeline (Reduced-Form First)

This folder contains the **structural extension** for the paper:
**"When Does Ability Grouping Improve Learning? Evidence from Three Experiments."**

The main paper remains reduced-form first. This structural block is a disciplined
counterfactual device, not the core identification engine. The current
blockwise version uses harmonized study-specific ITT targets, Kenya's
assignment-payoff diagnostic, class-size pressure scaling, and
shrinkage-regularized market residuals to discipline the production mapping:
the payoff to sorting is low at realized delivery fidelity, but can rise
nonlinearly when assignment-channel delivery becomes high.

The production block estimates a market-level ATE map, not a full individual
potential-outcome model:

```text
ATE_m(rho, omega, tau)
  = delta_m
  + lambda * rho * omega * tau^alpha
  + phi * (1 - tau) * (peer_shift_m + omega_r * rank_proxy_m)
  - chi_N * class_size_pressure_m
  - chi_V * grade_dispersion_m
```

The mechanical sorting first stage is fit separately as
`dispersion_m = sigma_eps_m + chi_V * (1 - omega_m) - kappa_sort * rho_m * omega_m`.
This separation is essential: Kenya has strong sorting compression but a near-zero
revealed assignment payoff at realized fidelity, so the model must not force the
dispersion first stage and achievement payoff to be the same parameter.

Parameter-level interpretation:
- `kappa_sort` is pinned primarily by the within-class dispersion moments.
- `lambda` and `tau_power` are jointly identified by market ITTs plus the Kenya
  assignment-payoff slope; Kenya's near-zero assignment payoff at realized
  fidelity prevents the model from putting a large payoff on correct placement
  at observed `tau`.
- `phi` and `omega_r` are anchored by peer/rank composites, but the rank
  component is weakly separated; interpret this as a composite social channel.
- `chi_N` and `chi_V` are cost-accounting terms for measured class-size pressure
  and grade-dispersion shifts, not independent class-size or grade-mixing
  experiments.

## Scripts

- `structural_blockwise_redesign.py`  
  Current canonical structural pipeline for paper use. It estimates the model in four blocks (`rho`, `omega`, `tau`, production), runs validation checks, writes counterfactual surfaces/summaries, writes pooled structural diagnostics, primitive-complementarity decompositions, and conditional stage-4 bootstrap intervals, and saves a `run_manifest.json` that defines which outputs belong to the current blockwise run.
  The current version also writes local stage-4 identification diagnostics, conditional production-parameter uncertainty, combined primitive-plus-production uncertainty, and a social-channel robustness check that re-estimates the production block with the rank weight fixed to zero.

- `verify_structural_package.py`  
  Post-estimation verification gate. It checks that the manifest and acceptance tests pass, every structural table used by the manuscript is generated by the canonical blockwise pipeline, stale legacy outputs are not cited, and the paper's main structural claims match the generated CSVs. It writes `output/structural_smm/structural_package_verification.json`.

- `empirical_moments.py`  
  Legacy all-at-once SMM helper. Builds empirical moments from cleaned market-level data and writes:
  - `output/structural_smm/empirical_moments.csv`
  - `output/structural_smm/empirical_moments_summary.csv`
  - `output/structural_smm/empirical_moments_todo.md`

- `model_primitives.py`  
  Legacy all-at-once SMM helper. Parameter transforms and parsimonious market-level structural mappings.

- `simulate_markets.py`  
  Legacy all-at-once SMM helper. Maps structural parameters into simulated moments and objective terms.

- `estimate_smm.py`  
  Legacy all-at-once SMM helper. Runs SMM optimization and saves:
  - `fitted_moments.csv`
  - `smm_parameter_table.csv`
  - `optimization_log.csv`
  - `worst_fit_moments_top20.csv`
  - `smm_run_summary.json`

- `bootstrap_counterfactuals.py`  
  Legacy all-at-once SMM helper. Produces bootstrap uncertainty and main counterfactuals:
  - missing high-rho/high-tau cell
  - designed vs realized Nigeria
  - one-at-a-time Nigeria decompositions

- `make_tables_figures.py`  
  Legacy all-at-once SMM helper. Produces paper-ready CSV tables and one decomposition figure.

## Run order

From repo root:

1. Run the canonical blockwise structural pipeline
```bash
python 3_Python/structural_blockwise_redesign.py
```

2. Verify that `output/structural_smm/run_manifest.json` lists the files used by the paper and that `acceptance_all_pass` is true.

3. Run the structural package verifier
```bash
python 3_Python/verify_structural_package.py
```

The verifier should report `Structural package verification: PASS` before the structural tables are copied into the paper.

Legacy all-at-once run order, not currently canonical for paper use:

1. Build moments
```bash
python 3_Python/empirical_moments.py
```

2. Estimate SMM (hardened first pass)
```bash
python 3_Python/estimate_smm.py --starts 12 --two-step --no-de
```

3. Bootstrap counterfactuals
```bash
python 3_Python/bootstrap_counterfactuals.py --n-boot 30 --starts 3
```

4. Build tables/figures
```bash
python 3_Python/make_tables_figures.py
```

## Identification and assumptions

What is tightly tied to data:
- Cross-market signal-quality ordering (`rho` moments).
- Treatment effects on dispersion/peer reallocation.
- Country-level ITT and peer/rank composite reduced-form moments.
- Kenya assignment-payoff slope.
- Nigeria implementation frictions (with both computed and external target moments).

What is weakly identified / functional-form sensitive:
- Level of `tau` by market and the nonlinear delivery-activation parameter.
- Counterfactual magnitudes for high-rho/high-tau scenarios.
- Decomposition shares across rho/tau/execution channels.
- The rank weight inside the peer/rank composite; the preferred interpretation is the composite social channel, not a separately identified rank effect.
- Market residual shrinkage prior and the common production-structure restriction.
- Combined primitive-plus-production uncertainty is approximate rather than a nonparametric bootstrap of the three original experiments.

## Guardrails

- Peer and rank are treated as a **composite social channel** (`zeta`), not separately identified.
- Nigeria assignment execution is modeled separately from raw signal quality.
- The estimable object is the market-level ATE map above; the structural section should not claim coefficient-by-coefficient identification of a richer micro production technology.
- Market residuals are shrinkage-regularized rather than allowed to mechanically interpolate country ITTs.
- Weak scale priors and hard comparative-static penalties are part of the preferred production estimator; the paper reports robustness when these restrictions are relaxed.
- Class-size shifts enter as class-size pressure in 25-student units.
- Structural section is an extension, not a replacement for reduced-form evidence.

In the current run, the main limitation is not the Kenya sorting first stage; that is now fit closely. Paper language should lean on assignment, delivery, process margins, and the high-input counterfactual rather than claiming precise identification of a standalone peer effect.
